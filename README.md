# playlist-maker-android-GavrilinaAnna
Учебный проект по созданию мобильного приложения

## О проекте

Playlist Maker - это мобильное приложение для поиска музыки, создания и управления персональными плейлистами.

## Текущий прогресс

### Спринт 3
- Создана базовая структура проекта
- Реализован главный экран приложения
- Добавлены экраны Search и Settings
- Настроена навигация между экранами

### Спринт 4
- Реализована навигация между экранами с использованием Jetpack Navigation
- Создан PlaylistHost с NavHost для управления переходами
- Доработан экран поиска:
    - Добавлена поисковая строка с иконками лупы и очистки
    - Реализована история поисковых запросов
    - Настроен интерфейс с заголовком и кнопкой назад
- Улучшен экран настроек:
    - Добавлен переключатель темы
    - Реализован функционал шеринга приложения
    - Настроена отправка email в поддержку
    - Добавлено открытие пользовательского соглашения
- Обновлены ресурсы приложения
- Улучшена архитектура с использованием единой MainActivity

### Спринт 5
Архитектурные изменения:
- Реорганизована структура проекта** по принципам Clean Architecture:
  - `data` слой - работа с данными и сетью
  - `domain` слой - бизнес-логика и модели
  - `ui` слой - пользовательский интерфейс
- Переименованы файлы экранов**: убрано "Activity" из названий в соответствии с Compose-подходом
- Создана новая структура пакетов**:
  - `ui/activity` - только MainActivity как контейнер
  - `ui/screen` - все Compose-экраны приложения
  - `ui/navigation` - навигационные компоненты
  - `creator` - управление зависимостями

### Спринт 6
- Создана SearchViewModel с управлением состояниями поиска:
  - первоначальное;
  - поиск идет;
  - поиск завершился успешно;
  - поиск завершился ошибкой;
- Обновлен экран поиска:
  - Добавлена логика поиска;
  - Добавлена интеграция с ViewModel;
  - Добавлена обработка состояний поиска;
  - Изменена логика отображения истории поиска (история показывается только в начальном состоянии при пустом запросе);
  - Реализовано отображение списка найденных треков;
  - Изменен UI поисковой строки (Иконка поиска вынесена в отдельный IconButton)
  - Добавлен обработчик клика на элементы истории поиска
  - Добавлены компоненты TrackListItem (для отображения треков) и SearchResultsList (для списка результатов)
- Исправлена архитектура (MainActivity -> MainScreeen, AppActivity)

### Спринт 8
- Реализована система навигации между главным экраном и экранами Songs, Playlists, Favorites;
- Реализованы экраны Избранного, где отображаются треки, добавленные в избранное;
- Реализован экран создания нового плейлиста. На экране два поля: для ввода имени плейлиста и для его описания, а также кнопка «Сохранить», при нажатии на которую созданный плейлист сохраняется в хранилище.
- На экране Playlists реализован FloatingActionButton. При нажатии на кнопку осуществляется навигация на экран создания нового плейлиста.
- Создан экран Track details с детальным описанием трека. Здесь размещены иконки «Добавить в избранное» и «Добавить в плейлист». При нажатии на «Добавить в избранное» у трека меняется значение favorite. При нажатии на «Добавить в плейлист» всплывает нижняя панель ModalBottomSheet со списком существующих плейлистов. При нажатии на плейлист в поле трека playlistId добавляется значение id выбранного плейлиста.
- Реализована история поиска вместо заглушки:
  - Каждый успешный поиск (по запросу была найдена хотя бы одна песня) добавляется в историю поиска;
  - Дубликаты в истории поиска удаляются;
  - Ограничение количества элементов истории поиска - 25, те хранятся последние 25 успешных запросов.
