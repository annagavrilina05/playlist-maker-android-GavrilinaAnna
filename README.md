# playlist-maker-android-GavrilinaAnna
Учебный проект по созданию мобильного приложения

## О проекте

Playlist Maker - это мобильное приложение для поиска музыки, создания и управления персональными плейлистами.

## Текущий прогресс

### Спринт 3
- Создана базовая структура проекта
- Реализован главный экран приложения
- Добавлены экраны Search и Settings
- Настроена навигация между экранами

### Спринт 4
- Реализована навигация между экранами с использованием Jetpack Navigation
- Создан PlaylistHost с NavHost для управления переходами
- Доработан экран поиска:
    - Добавлена поисковая строка с иконками лупы и очистки
    - Реализована история поисковых запросов
    - Настроен интерфейс с заголовком и кнопкой назад
- Улучшен экран настроек:
    - Добавлен переключатель темы
    - Реализован функционал шеринга приложения
    - Настроена отправка email в поддержку
    - Добавлено открытие пользовательского соглашения
- Обновлены ресурсы приложения
- Улучшена архитектура с использованием единой MainActivity

### Спринт 5
Архитектурные изменения:
- Реорганизована структура проекта** по принципам Clean Architecture:
  - `data` слой - работа с данными и сетью
  - `domain` слой - бизнес-логика и модели
  - `ui` слой - пользовательский интерфейс
- Переименованы файлы экранов**: убрано "Activity" из названий в соответствии с Compose-подходом
- Создана новая структура пакетов**:
  - `ui/activity` - только MainActivity как контейнер
  - `ui/screen` - все Compose-экраны приложения
  - `ui/navigation` - навигационные компоненты
  - `creator` - управление зависимостями

### Спринт 6
- Создана SearchViewModel с управлением состояниями поиска:
  - первоначальное;
  - поиск идет;
  - поиск завершился успешно;
  - поиск завершился ошибкой;
- Обновлен экран поиска:
  - Добавлена логика поиска;
  - Добавлена интеграция с ViewModel;
  - Добавлена обработка состояний поиска;
  - Изменена логика отображения истории поиска (история показывается только в начальном состоянии при пустом запросе);
  - Реализовано отображение списка найденных треков;
  - Изменен UI поисковой строки (Иконка поиска вынесена в отдельный IconButton)
  - Добавлен обработчик клика на элементы истории поиска
  - Добавлены компоненты TrackListItem (для отображения треков) и SearchResultsList (для списка результатов)
- Исправлена архитектура (MainActivity -> MainScreeen, AppActivity)

### Спринт 8
- Реализована система навигации между главным экраном и экранами Songs, Playlists, Favorites;
- Реализованы экраны Избранного, где отображаются треки, добавленные в избранное;
- Реализован экран создания нового плейлиста. На экране два поля: для ввода имени плейлиста и для его описания, а также кнопка «Сохранить», при нажатии на которую созданный плейлист сохраняется в хранилище.
- На экране Playlists реализован FloatingActionButton. При нажатии на кнопку осуществляется навигация на экран создания нового плейлиста.
- Создан экран Track details с детальным описанием трека. Здесь размещены иконки «Добавить в избранное» и «Добавить в плейлист». При нажатии на «Добавить в избранное» у трека меняется значение favorite. При нажатии на «Добавить в плейлист» всплывает нижняя панель ModalBottomSheet со списком существующих плейлистов. При нажатии на плейлист в поле трека playlistId добавляется значение id выбранного плейлиста.
- Реализована история поиска вместо заглушки:
  - Каждый успешный поиск (по запросу была найдена хотя бы одна песня) добавляется в историю поиска;
  - Дубликаты в истории поиска удаляются;
  - Ограничение количества элементов истории поиска - 25, те хранятся последние 25 успешных запросов.

### Спринт 9
- Создан экран плейлиста;
- Осуществлен переход с передачи полных объектов Track между экранами на передачу идентификаторов (ID) треков;
- Устранено дублирование кода: Удалена дублирующая функция  TrackListItem из экрана избранного, теперь используется единый компонент из экрана поиска;
- Улучшение интерфейса:
  - Реализовано отображение обложек треков при помощи интеграции библиотеки Coil для асинхронной загрузки изображений. В случае отстутствия обложки или её загрузки отображается заглушка;
  - Добавлены недостающие изображения на уже существующие экраны согласно дизайн-макету Figma;
  - Исправлен внешний вид экрана деталей трека на приведенный в макете Figma;
  - Исправлена проблема с BottomSheet на экране деталей трека:
    - Устранена проблема с отображением только первого элемента списка плейлистов;
    - Реализована полноценная прокрутка списка;
    - Реализована корректная работа для вытягивания/скрытия панели;
- Добавлена функциональность поиску:
  - Интеграция с iTunes API: Реализован полноценный поиск треков через официальный iTunes Search API;
  - Улучшения:
    - Поднятие интерфейсных элементов выше для видимости при открытой клавиатуре;
    - Автоматическое скрытие клавиатуры при на жатие на крестик или на лупу в строке поиска;
- Добавлена обработка склонений слова "треки" в приложении;
- Исправлено отображение описания плейлистов: теперь точка отображается перед описанием плейлиста только при наличии этого описания;
- Исправлена обработка ошибок:
  - Решена проблема с недоступностью экрана ошибки "Проблемы со связью";
  - Добавлена кнопка "Обновить" на экран ошибки "Проблемы со связью", которая повторяет последний неудавшийся поисковый запрос;
- Добавлена иконка приложения;

### Спринт 10
- Реализовано хранение плейлистов в базе данных:
  - Заменено использование моков на реальную базу данных Room во всех методах PlaylistsRepositoryImpl и TracksRepositoryImpl;
  - Обеспечено сохранение созданных плейлистов и треков между запусками приложения;
- Интеграция DataStore для хранения истории поиска:
  - Создан класс SearchHistoryPreferences для работы с DataStore;
  - Реализовано добавление поисковых запросов с учётом ограничений:
    - Сохранение не более 10 запросов;
    - Устранение дубликатов;
    - Хранение запросов в обратном хронологическом порядке (самый новый — первый);
  - Обновлён SearchHistoryRepositoryImpl для работы с SearchHistoryPreferences;
  - Удалён DatabaseMock;

### Спринт 12
- Реализована темная тема для всего приложения:
  - Созданы цветовые схемы для светлой и темной темы в Theme.kt;
  - Обновлены все экраны для поддержки динамической смены темы;
  - Использован ThemeViewModel и ThemePreferences для сохранения выбора пользователя;
  - На экране настроек добавлен переключатель темы, который сохраняет состояние в DataStore;
  - На экранах ошибок поиска ("ничего не найдено", "ошибка соединения") и экране пустого избранного теперь отображаются разные картинки в зависимости от выбранной темы (светлая/темная);
- Экран избранного: добавлено удаление трека по долгому нажатию на элемент списка;
- Удален дублирующий TrackListItem из экрана плейлиста, используется единый компонент из экрана поиска;
- Реализована дополнительная фича: Удаление плейлиста (Вариант 1):
  - Удаление из деталей плейлиста: добавлена иконка корзины на экране плейлиста, при нажатии происходит удаление с возвратом на список плейлистов;
  - Удаление из списка: реализовано удаление по долгому нажатию на элемент в списке плейлистов;
  - После удаления пользователь автоматически возвращается на экран списка плейлистов;
  - Добавлены Toast-уведомления при удалении плейлистов.
